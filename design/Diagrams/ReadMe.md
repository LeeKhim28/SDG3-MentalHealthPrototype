## Use Case Diagram Explanation
The MindCare use case diagram illustrates interactions between three actors: Users, Therapists, and the Payment System. Users must register before they can log in, and most actions—such as managing accounts, chatting with the assistant, recording moods, viewing mood insights, connecting with therapists, joining sessions, and performing wind-down activities—require authentication. Some use cases use generalization, such as counselling sessions (video or text) and wind-down activities (breathing exercises, music, sleep stories, tips), providing flexibility in achieving goals. Exclude relationships exist where recording moods can trigger optional features like mood insights or assistant suggestions. Therapists manage session availability and conduct sessions (video or text), while the Payment System handles session payments, which are mandatory for booking. Some therapist and payment features are planned for future implementation. Overall, the diagram represents a structured, flexible system supporting mental well-being.

## Class Diagram Explanation
The MindCare system supports mental health through counseling, mood tracking, and relaxation activities. The User class is central, with methods for login, profile management, mood tracking, requesting counseling sessions, and starting wind-down activities. UserWindDownActivity connects to specific types such as BreathingExercise, CalmingAudio, SleepStory, and SmartWindDownTips, allowing users to interact with each activity. MoodRecord stores users’ emotional data, which links to MoodInsight to generate summaries of stress, sleep, and emotional patterns.

Counseling sessions are handled through the Event class. Users are linked to events, which are associated with therapists. Event is specialized into Appointment and CounsellingSession, and CounsellingSession is further divided into Chat_Session (using Chat_Message objects) and VideoCallSession (with camera and microphone controls). The Therapist class manages availability, session scheduling, and ratings. UserSettings allows control over privacy and data-sharing preferences. Overall, the class diagram illustrates the system’s structure, relationships, and functionalities through its methods.
